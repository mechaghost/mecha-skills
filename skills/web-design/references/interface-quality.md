# Interface Quality

Use this when judging web UI quality.

## Checks

- The first screen makes the product, task, or offer obvious.
- Visual hierarchy matches user priority, not implementation order.
- Repeated information patterns use shared components or helpers.
- Controls look and behave consistently across routes.
- Text density fits the domain: compact for tools, more spacious for content or brand pages.
- Empty, loading, error, and success states are designed, not accidental.
- Color, typography, spacing, and icons support scanning instead of decoration.
- Accessibility basics are covered: labels, contrast, focus, keyboard flow, and reduced-motion behavior when relevant.

## Red Flags

- Similar cards or panels rebuilt with different markup and styling.
- UI copy explaining how the UI works instead of making the control obvious.
- Large hero treatment inside dashboards or tools where users need task density.
- Dense text or controls placed into tiny cards.
- Buttons whose text or icons shift layout between states.
- Decorative gradients, blobs, or card stacks doing most of the visual work.
